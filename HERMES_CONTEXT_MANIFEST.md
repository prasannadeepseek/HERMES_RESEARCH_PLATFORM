# 📜 Hermes Context Manifest (v1.0)

> **Role:** You are an expert Quant Developer assisting with the **Hermes Research Platform**.
> **Objective:** Maintain a modular, agentic trading strategy generation loop using `vectorbt` and `OpenAlgo`.

## 1. System Map (`repo_structure.json`)

- **`/agent`**: Orchestration logic. `runner.py` is the entry point.
- **`/backtester`**: Evaluation engine. `engine.py` handles `vectorbt` and OASIS stress tests.
- **`/data_pipeline`**: Connector logic. `openalgo_connector.py` handles all external REST calls.
- **`/hermes_wiki`**: Persistence layer for LLM insights and strategy performance history.

## 2. Core Class Interfaces (LLD)

### A. The Strategy Interface (LLM Target)

```python
    Returns: (entries, exits, short_entries, short_exits) as Boolean Series
    """
    # Logic here...
    return entries, exits, None, None

PARAM_RANGES = {
    "window": range(10, 50, 5),
    "std_dev": [1.5, 2.0, 2.5]
}
```

### B. `HermesRunner` (agent/runner.py)

- `execute_research_loop(iterations)`: The main `while` loop.
- `_run_strategy(code, df, params)`: Executes generated code using `exec()` in a restricted namespace.
- `_sanitize_code(code)`: Regex-based security filter.

### C. `HermesBacktester` (backtester/engine.py)

- `evaluate_signals(...)`: Wraps `vbt.Portfolio.from_signals`.
- `run_oasis_stress_test(...)`: Tests strategy against `["volatile", "crash", "trending"]` synthetic data.
- `check_goals(metrics, config)`: Boolean check if ROI/Drawdown meet user requirements.

### D. `OpenAlgoClient` (data_pipeline/openalgo_connector.py)

- `get_historical_data(symbol, interval, n_bars)`: Returns cleaned OHLCV DataFrame.
- `place_smart_order(symbol, side, quantity)`: Standardized execution via OpenAlgo REST API.

## 3. Interaction Protocols

| Task                | Action                                                                        |
| :------------------ | :---------------------------------------------------------------------------- |
| **New Strategy**    | Use `agent/templates` to guide the LLM; must include `PARAM_RANGES`.          |
| **Optimization**    | Use `HermesOptimizer` to iterate through `PARAM_RANGES` before OASIS.         |
| **Saving Progress** | Write MD files to `hermes_wiki/` using `Session_{Symbol}_{Timestamp}` format. |
| **Execution**       | Convert signals to `OpenAlgo` orders only if `OASIS_SCORE > 70`.              |

## 4. Technical Constraints

- **Backtester:** Uses `vectorbt` (vbt), not `backtrader`.
- **Data:** Pandas DataFrames with lower-case columns `['open', 'high', 'low', 'close', 'volume']`.
- **API:** All broker communication goes through `http://localhost:5001` (OpenAlgo Gateway).
- **Environment:** Python 3.10+, Dockerized.
